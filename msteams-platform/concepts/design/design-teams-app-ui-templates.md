# <a name="designing-your-microsoft-teams-app-with-ui-templates"></a>Conception de votre application Microsoft teams avec des modèles d’interface utilisateur

Concevez plus rapidement votre application Microsoft teams avec les modèles d’interface utilisateur. Les modèles sont une collection de composants basés sur l’interface utilisateur Fluent qui fonctionnent dans des cas d’utilisation de teams communs, ce qui vous permet de déterminer la meilleure expérience pour vos utilisateurs.

## <a name="microsoft-teams-ui-kit"></a>Kit d’interface utilisateur Microsoft teams

Vous pouvez obtenir des modèles d’interface utilisateur à partir du kit d’interface utilisateur Microsoft Teams, qui contient également des informations détaillées sur l’utilisation, l’anatomie, l’accessibilité et les meilleures pratiques.

> [!div class="nextstepaction"]
> [Obtenir le kit d’interface utilisateur Microsoft Teams (Figma)](https://www.figma.com/community/file/916836509871353159)

## <a name="list"></a>Répertorier

Vous pouvez utiliser une liste pour afficher les éléments associés dans un format pouvant être analysé et permettre aux utilisateurs d’effectuer des actions sur une liste entière ou des éléments individuels.

:::image type="content" source="../../assets/images/ui-templates/list.png" alt-text="Exemple de modèle d’interface utilisateur de liste." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Afficher les données
* Actions contextuelles sur le contenu de l’application

## <a name="dashboard"></a>Tableau de bord

Un tableau de bord affiche différents types de contenu dans un emplacement central (application personnelle ou onglet Teams). Les utilisateurs doivent pouvoir personnaliser au moins une partie de ce qu’ils voient sur un tableau de bord.

:::image type="content" source="../../assets/images/ui-templates/dashboard.png" alt-text="Exemple affiche un modèle d’interface utilisateur de tableau de bord." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Analyser les données
* Mesures du rapport
* Organiser différentes informations à un seul endroit

## <a name="form"></a>Formulaire

Les formulaires sont utilisés pour collecter, valider et soumettre l’entrée de l’utilisateur de manière structurée. L’étiquetage clair et les regroupements logiques de champs d’entrée sont essentiels pour une expérience utilisateur optimale.

:::image type="content" source="../../assets/images/ui-templates/form.png" alt-text="Exemple de modèle d’interface utilisateur de formulaire." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Se connecter
* Profils utilisateur
* Paramètres
* Collection d’entrée utilisateur

## <a name="sign-in"></a>Se connecter

Vous pouvez concevoir des flux de connexion d’application pour différents contextes de teams et fournisseurs d’identité. L’exemple suivant inclut l’authentification unique (SSO), que nous recommandons pour l’expérience d’authentification la plus simple.

:::image type="content" source="../../assets/images/ui-templates/sign-in.png" alt-text="Exemple affiche un modèle d’interface utilisateur de connexion." border="false":::

### <a name="top-use-case"></a>Cas d’utilisation le plus fréquent

* Authentifier les utilisateurs

## <a name="task-board"></a>Tableau des tâches

Un tableau des tâches, parfois appelé tableau kanban ou couloirs de natation, est une collection de cartes souvent utilisées pour suivre l’état des éléments de travail ou des tickets. Elle peut également être utilisée pour trier tout type de contenu en catégories. Vous pouvez modifier et déplacer les cartes entre les colonnes.

:::image type="content" source="../../assets/images/ui-templates/task-board.png" alt-text="Exemple de modèle d’interface utilisateur de tableau des tâches." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Gestion de projet. Affectation des tâches et du suivi de l’État
* Brainstorming. Ajout d’idées dans différentes catégories
* Les exercices de tri. Organisation de tous les types d’informations en compartiments

## <a name="data-visualization"></a>Visualisation de données

Vous pouvez utiliser différentes tailles de carte (simple, double et complet) pour empiler et organiser les visualisations de données sur la même page. Les cartes s’ajustent pour s’ajuster à la disposition des colonnes et remplir les espaces vides.

:::image type="content" source="../../assets/images/ui-templates/data-viz.png" alt-text="Exemple de modèle d’interface utilisateur de visualisation des données." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Afficher des informations complexes
* Créer un tableau de bord

## <a name="wizard"></a>Assistant

Un Assistant guide un utilisateur à travers plusieurs écrans pour effectuer une tâche (par exemple, un processus de configuration).

:::image type="content" source="../../assets/images/ui-templates/wizard.png" alt-text="Exemple affiche un modèle d’interface utilisateur de l’Assistant." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Configuration
* Intégration
* Expériences de première exécution

## <a name="empty-state"></a>État vide

Le modèle d’état vide peut être utilisé pour de nombreux scénarios, y compris la connexion, les expériences de première exécution, les messages d’erreur, et bien plus encore. Elle est très flexible : elle s’adapte pour utiliser un, plusieurs ou tous les composants de la conception suivante.

:::image type="content" source="../../assets/images/ui-templates/empty-state.png" alt-text="Exemple affiche un modèle d’interface utilisateur de l’Assistant." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Se connecter
* Messages de bienvenue et expériences de première exécution
* Messages de réussite
* Messages d’erreur

## <a name="notification-bar"></a>Barre de notification

Une barre de notification est une zone dédiée pour afficher un bref message important qui ne demande pas à l’utilisateur de prendre des mesures immédiates. Des couleurs d’arrière-plan et des icônes spécifiques sont associées à des types de messages spécifiques (voir ci-dessous).

:::image type="content" source="../../assets/images/ui-templates/notification-bar.png" alt-text="Exemple affiche les modèles de barre de notification." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Messages, erreurs et avertissements critiques
* Messages de réussite
* Messages d’information ou de promotion

## <a name="left-nav"></a>Navigation de gauche

Utilisez le panneau de navigation gauche pour parcourir plusieurs pages au sein de votre onglet Teams. Dans l’exemple suivant, la navigation de gauche se trouve entre la liste de canaux et le contenu de tabulation.

:::image type="content" source="../../assets/images/ui-templates/left-nav.png" alt-text="Exemple affiche un modèle de navigation gauche." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Parcourir plusieurs pages dans un onglet teams
* Décomposer les applications complexes en plusieurs pages

## <a name="breadcrumb"></a>Barre de navigation

Les barres de navigation sont des aides à la navigation qui transmettent la hiérarchie de votre application. Ils aident les utilisateurs à comprendre comment la page qu’ils affichent dans l’expérience globale et à accéder en un clic aux niveaux supérieurs de cette hiérarchie.

:::image type="content" source="../../assets/images/ui-templates/breadcrumb.png" alt-text="Exemple affiche un modèle de barre de navigation." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Communication de la hiérarchie
* Navigation

## <a name="toolbar"></a>Barre d'outils

Une barre d’outils est un conteneur qui permet de regrouper un ensemble de contrôles.

:::image type="content" source="../../assets/images/ui-templates/toolbar.png" alt-text="Exemple montre un modèle de barre d’outils." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Actions contextuelles sur le contenu de l’application
* Filtre contextuel et Rechercher
* Navigation et arborescences hiérarchiques

## <a name="stage"></a>Phase

Stage offre un moyen aux utilisateurs d’ouvrir une entité, comme une image, un fichier ou un site Web, dans teams au lieu de l’ouvrir dans une autre application ou un autre navigateur. Le cas d’utilisation principale de la phase est l’affichage ; la surface ne doit pas être utilisée pour les interactions complexes.

(Mise en œuvre Remarque : créez votre étape à l’aide d’un [module de tâches](../../task-modules-and-cards/task-modules/design-teams-task-modules.md)volumineux.)

:::image type="content" source="../../assets/images/ui-templates/stage.png" alt-text="Exemple de modèle stage." border="false":::

### <a name="top-use-cases"></a>Cas d’utilisation les plus fréquents

* Ouvrir une entité dans teams au lieu d’une autre application ou d’un autre navigateur
* Média Gong ou autre contenu
