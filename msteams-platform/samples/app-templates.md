# <a name="app-templates-for-microsoft-teams"></a>Modèles d’application pour Microsoft Teams

Les modèles d’application sont des applications Microsoft Teams prêtes pour la production qui sont impulsées par la communauté, open source et disponibles sur GitHub. Chacune contient des instructions détaillées de déploiement et d’installation pour votre organisation, offrant ainsi une application prête à l’emploi installable et utilisable immédiatement. Il fournit également un exemple d’application que vous pouvez installer et commencer à utiliser immédiatement. Le code source complet est également disponible, de sorte que vous pouvez l’explorer en détail ou répliquer le code et le modifier en fonction de vos besoins spécifiques.
Tous les modèles d’application sont fournis selon les termes du [contrat de licence MIT](https://github.com/OfficeDev/microsoft-teams-apps-eprescription/blob/master/LICENSE) .

> [!NOTE]
> Vous devez concéder des licences et la prise en charge des applications créées à partir de modèles d’application pour vos utilisateurs et organisations.

Les principaux avantages de l’utilisation de modèles d’application sont les suivants :

* **Déployer directement dans le cloud :** Tous les modèles d’application incluent des scripts de déploiement qui vous permettent d’héberger tous les services requis dans Microsoft Azure ou Power Platform.
* **Exemple de code recommandé :** Les modèles d’application sont conformes aux meilleures pratiques recommandées en matière de sécurité et d’infrastructure. Toutes les modifications soumises par la communauté aux modèles d’application sont examinées pour garantir la conformité.
* **Personnalisable et extensible :** Bien que tous les modèles d’application soient déployés avec une configuration minimale, l’ensemble des scripts de base de code et de déploiement sont fournis, afin que vous puissiez facilement les personnaliser ou les étendre en fonction de vos besoins uniques.
* **Documentation détaillée :** Tous les modèles d’application sont accompagnés d’une documentation de bout en bout sur l’architecture de la solution, le déploiement et les étapes de configuration.  

## <a name="adoption-tool--champion-management-platform"></a>Outil d’adoption - Plateforme de gestion champion

Le modèle d’application CMP (Champion Management Platform) vous aide à gérer, mettre à l’échelle et inspirer vos champions du travail d’équipe pour obtenir plus. Ce modèle d’application est basé sur le SharePoint Framework et chargé dans un onglet au sein d’une équipe. Les groupes peuvent exploiter cet outil pour faciliter la gestion des membres du programme, fournir un tableau de classement et des types d'événements pour l'enregistrement, ainsi que des outils pour superposer des badges numériques aux participants du programme.

[Obtenez-le sur GitHub](https://github.com/OfficeDev/microsoft-teams-apps-champion-management)

## <a name="adoption-tool--microsoft-365-learning-pathways-get-started"></a>Adoption Tool- Microsoft 365 Learning Pathways (Prise en main)

Le modèle d’application Prise en main vous permet d’apporter la puissance de Microsoft 365 parcours d’apprentissage à l’intérieur de Microsoft Teams. Ce modèle d'application vous permet d'accorder un accès facile à des pages de formation spécifiques ou à d'autres ressources intranet et de charger le contenu directement dans Teams. Vous pouvez également modifier le nom ou le logo de l’application pour qu’il corresponde à la marque de votre entreprise.

[Obtenez-le sur GitHub](https://github.com/msft-teams/tools/tree/master/M365%20Learning%20Pathways)

## <a name="company-communicator"></a>Communicateur d’entreprise

L’application Communicator d’entreprise permet aux équipes d’entreprise de créer et d’envoyer des messages destinés à plusieurs équipes ou à un grand nombre d’employés via une conversation, ce qui permet à l’organisation d’atteindre les employés là où ils collaborent. Utilisez ce modèle pour plusieurs scénarios tels que les nouvelles annonces d’initiative, l’intégration des employés, l’apprentissage moderne et le développement ou les diffusions à l’échelle de l’organisation.

L’application fournit une interface simple permettant aux utilisateurs désignés de créer, d’afficher un aperçu, de collaborer et d’envoyer des messages.

Il fournit une base pour créer des fonctionnalités de communication ciblées personnalisées telles que la télémétrie personnalisée sur le nombre d’utilisateurs reconnus ou ayant interagi avec un message.

[Obtenez-le sur GitHub](https://github.com/OfficeDev/microsoft-teams-company-communicator-app)

## <a name="co-worker-appreciation"></a>Appréciation du collègue

En utilisant le modèle d'appréciation des collègues dans Microsoft Teams, les utilisateurs peuvent reconnaître les réalisations de leurs collègues dans le contexte de Teams. Lorsque les collègues choisissent de récompenser un collègue, les destinataires et les autres membres de l’équipe sont marqués dans une conversation de canal et reçoivent une notification sur les détails du prix du canal. Les récompenses sont enregistrées dans l’application Teams, qui est sécurisée, portable et facile à partager. Il s’agit de la version PowerApps du modèle d’application Open Badges, avec un classement.

## <a name="faq-plus"></a>Forum aux questions

Les bots Q&A conversationnels sont un moyen simple de fournir des réponses aux questions fréquemment posées par les utilisateurs. Toutefois, la plupart des bots ne parviennent pas à interagir avec les utilisateurs de manière significative, car il n’y a aucun humain dans la boucle lorsque le bot échoue. Le bot FAQ est un bot Q&Un bot convivial qui apporte un humain dans la boucle lorsqu’il ne peut pas vous aider. On peut poser une question au bot et le bot répond avec une réponse s’il est contenu dans le base de connaissances. Si ce n’est pas le cas, le bot permet à l’utilisateur d’envoyer une requête, qui est ensuite publiée dans une équipe préconfigurée d’experts qui aident à fournir un support en agissant sur les notifications de l’équipe elle-même.

> [!NOTE]
> La dernière version de **FAQ Plus** prend en charge les résolutions Q&A améliorées en permettant à une équipe d’experts d’effectuer les opérations suivantes :
>
> &#x2714; Ajouter de nouveaux&Q directement au base de connaissances à l’aide d’extensions de message.
>
> &#x2714; modifier et supprimer les paires Q&A ajoutées par un bot.
>
> &#x2714; Suivre l’historique des révisions de Q&As.
>
> &#x2714; Configurer une réponse avec des détails supplémentaires à afficher en tant que [carte adaptative](../task-modules-and-cards/cards/cards-reference.md#adaptive-card).
>
[Obtenez-le sur GitHub](https://github.com/OfficeDev/microsoft-teams-apps-faqplusv2)

## <a name="icebreaker"></a>Brise-glace

Icebreaker est un [bot Microsoft Teams](../bots/what-are-bots.md) qui aide votre équipe à se rapprocher en associant deux membres d’équipe aléatoires chaque semaine pour se rencontrer. Le bot facilite la planification en suggérant automatiquement des heures gratuites qui fonctionnent pour les deux membres. Renforcez les connexions personnelles et créez une communauté étroitement liée avec cette application.

En plus d’encourager les connexions personnelles au sein de toute votre équipe, l’application Icebreaker peut aider à cultiver des communautés basées sur les centres d’intérêt au sein de votre organisation. Par exemple, vous pouvez utiliser cette application pour un groupe d’intérêt DevOps afin d’aider les idées et les meilleures pratiques à se répartir de manière organique dans votre organisation.

[Obtenez-le sur GitHub](https://github.com/OfficeDev/microsoft-teams-icebreaker-app)
